# ARCHITECTURE

## Слои

- `src/data` хранит карты, главы и сценарии.
- `src/domain` собирает трактовки, прогресс и storage.
- `src/app.ts` рендерит интерфейс и обрабатывает действия.
- `src/style.css` отвечает за мобильную визуальную оболочку.
- `src/main.ts` монтирует приложение в DOM.

## Доменная модель

- `player state` хранит опыт, текущую главу, текущую ситуацию и последний выбор.
- `card model` описывает архетип и его значения.
- `story chapter model` задаёт мост между картой и сценой.
- `encounter model` описывает конкретную ситуацию с выбором.
- `progress helpers` считают уровень и собирают карточку прогресса.
- `storage helpers` читают и пишут `localStorage`.
- `random helpers` оставлены как задел для будущих сцен, но не управляют PR-000.

## Принцип трактовки

Значение собирается композиционно:

- базовая карта;
- контекст главы или ситуации;
- ориентация;
- точечные overrides для конкретного выбора.

Это позволяет не дублировать полный текст каждой карты в каждом экране.

## Ограничения

- Без backend.
- Без внешних API.
- Без серверного хранения.
- Без авторизации.
- Без тяжёлых зависимостей.
