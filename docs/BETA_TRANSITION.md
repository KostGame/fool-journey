# BETA_TRANSITION

Базовая спецификация перехода от alpha к первой beta-версии мобильного текстового квеста `Путь Шута`.
Источник концепции: Issue #31.

## Цель

Провести проект по цепочке `SU-011 → SU-014` без большого объединённого PR.
Каждый шаг живёт в отдельной feature-ветке, получает отдельный task-файл, отдельный report-файл и обычный Open PR.

## Что считается beta

- mobile-first flow без горизонтальной прокрутки;
- экранная навигация `home → scene → result → next scene → completion`;
- понятные вопросы, действия и ответы;
- `Дневник Шута` и inventory state в `localStorage`;
- локальные миниатюры карт и fallback для отсутствующих изображений;
- корректная кодировка русских строк;
- статический хостинг на GitHub Pages без backend и внешних API.

## Последовательность работ

1. `SU-011` - язык сцен и результатов под мобильный text quest.
2. `SU-012` - локальные миниатюры карт Таро.
3. `SU-013` - аудит кодировки и качества контента, прямые вопросы и чистые кнопки.
4. `SU-014` - финальный beta smoke pass.

## Правила цепочки

- Каждая задача идёт в своей ветке от актуального `main`.
- PR открывается как обычный Open PR, не Draft.
- После зелёного CI PR можно мержить только если это отдельно разрешено в задаче.
- Если найден крупный блокер, его честно фиксируют в report и не заявляют beta-ready.
- После merge одного SU следующий SU стартует только от актуального `main`.

## Текущий фокус

- текущий beta-шаг: `SU-013`;
- в работе аудит кодировки и качества текста;
- источник ассетов задокументирован в [docs/ASSETS.md](docs/ASSETS.md);
