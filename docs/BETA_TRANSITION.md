# BETA_TRANSITION

Базовая спецификация перехода от alpha к первой beta-версии мобильного текстового квеста `Путь Шута`.
Источник концепции: Issue #31.

## Цель

Довести проект до первой beta-версии без большого объединённого PR:

- каждый SU живёт в отдельной ветке;
- каждый SU получает отдельный task-файл;
- каждый SU получает отдельный report-файл;
- Codex открывает обычный Open PR, не Draft;
- после merge одного SU следующий начинается только от актуального `main`.

## Что считается beta

- mobile-first flow без горизонтальной прокрутки;
- экранная навигация `home → scene → result → next scene → completion`;
- понятные вопросы, действия и ответы;
- `Дневник Шута` и inventory state в `localStorage`;
- локальные thumbnails карт и fallback для отсутствующих изображений;
- корректная кодировка русских строк;
- статический хостинг на GitHub Pages без backend и внешних API.

## Итог цепочки

1. `SU-011` - язык сцен и результатов под мобильный text quest.
2. `SU-012` - локальные thumbnails карт Таро.
3. `SU-013` - аудит кодировки и качества контента.
4. `SU-014` - финальный beta smoke pass.

Цепочка завершена. `Project status: beta`.

## Правила

- каждая задача идёт от актуального `main`;
- PR открывается как обычный Open PR, не Draft;
- визуальные и мобильные блокеры честно фиксируются в отчёте;
- если найден крупный blocker, он выносится в follow-up Issue;
- beta-ready заявляется только после проверки Pages, журнала, thumbnails, инвентаря, completion и кодировки.

## Текущий ориентир

Перейти от альфа-прототипа к устойчивой beta-версии можно только при выполнении чек-листа в [docs/BETA_CHECKLIST.md](docs/BETA_CHECKLIST.md).
