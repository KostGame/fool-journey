# BETA_TRANSITION

Базовая спецификация перехода от alpha к первой beta-версии мобильного текстового квеста `Путь Шута`.
Источник концепции: Issue #31.

## Цель

Провести проект по цепочке `SU-011 → SU-014` без большого объединённого PR.
Каждый шаг оформляется отдельной веткой, отдельным task-файлом, отдельным report-файлом и обычным Open PR.

## Что считается beta

- мобильный first-flow без горизонтальной прокрутки;
- screen-by-screen навигация `home → scene → result → next scene → completion`;
- понятные вопросы, действия и ответы;
- дневник Шута и inventory state на `localStorage`;
- миниатюры карт и fallback для отсутствующих изображений;
- проверенная кодировка русских строк;
- статический хостинг на GitHub Pages без backend и внешних API.

## Последовательность работ

1. `SU-011` - перестроить язык UI сцен и результатов под нормальный mobile text quest.
2. `SU-012` - добавить миниатюры карт и локальные ассеты.
3. `SU-013` - проверить кодировку и качество игровых текстов.
4. `SU-014` - финальный mobile smoke pass и beta readiness.

## Правила цепочки

- каждая задача живёт в своей feature-ветке;
- у каждой задачи есть свой `tasks/SU/*.md`;
- у каждой задачи есть свой `reports/*.md`;
- PR открывается как обычный Open PR, не Draft;
- после зелёного CI можно мержить только если это отдельно разрешено;
- если найден крупный блокер, его честно фиксируют в отчёте и beta-ready не заявляют;
- после merge одного SU следующий SU стартует только от актуального `main`.

## Текущий фокус

- `SU-011` меняет язык экранов сцены и результата;
- `SU-011` не добавляет новые карты, главы или режимы;
- `SU-011` не ломает `journal` и основной story flow;
- `SU-011` должен приблизить интерфейс к читаемому мобильному текстовому квесту.

## Критерии beta readiness

- основная история запускается на телефоне без длинной активной страницы;
- сцены читаются как диалоги, а не как технический прототип;
- выборы выглядят как действия Шута;
- дневник и инвентарь сохраняются локально;
- Pages открывается стабильно и показывает русские строки корректно;
- проверки `npm run check`, `npm test`, `npm run build` и `git diff --check` проходят.
