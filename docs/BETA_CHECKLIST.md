# BETA_CHECKLIST

## Цель

Этот чек-лист нужен для финальной проверки beta-готовности `Путь Шута` на GitHub Pages.

## Обязательные проверки

- открыть Pages URL;
- проверить home screen;
- проверить scene screen;
- проверить result screen;
- проверить `Дневник Шута`;
- проверить completion screen;
- проверить inventory statuses `Получено`, `Применено`, `Помощник`;
- проверить thumbnails и fallback;
- проверить mobile viewport 360-430 px;
- проверить отсутствие горизонтальной прокрутки;
- проверить отсутствие `U+FFFD` и явных mojibake-паттернов;
- запустить `npm run check:encoding`;
- запустить `npm run check`;
- запустить `npm test`;
- запустить `npm run build`;
- запустить `git diff --check`.

## Ручной smoke-test

1. Открыть `https://kostgame.github.io/fool-journey/`.
2. Начать историю через `Продолжить историю`.
3. Пройти несколько сцен и убедиться, что flow идёт как `home → scene → result → next scene`.
4. Проверить, что на сцене есть прямой вопрос и 2-3 понятных выбора.
5. Проверить, что на result screen видны выбранное действие, реакция и кнопка `Дальше`.
6. Открыть `Дневник Шута` и убедиться, что журнал показывает следы пути, inventory и thumbnails.
7. Дойти до completion screen и проверить, что replay работает.
8. Перезагрузить страницу и убедиться, что прогресс восстанавливается из `localStorage`.
9. На ширине 360-430 px убедиться, что ничего не налезает и не появляется горизонтальная прокрутка.

## Beta verdict

Beta-ready можно считать только тогда, когда все проверки выше проходят без визуальных и текстовых блокеров.
