# Путь Шута

Статический браузерный прототип про путь старших арканов Таро от Шута к Миру. Между большими главами появляются короткие дорожные события младших арканов.

## Статус

Текущая линия разработки: `SU-003` - первый слой minor events поверх полного пути старших арканов.

## Что уже есть

- Vite + TypeScript + Vitest.
- Главный экран на русском языке.
- Первый играбельный путь Шута.
- Данные для 22 старших арканов и всех 56 младших арканов.
- Короткие дорожные события между большими главами.
- Сохранение прогресса в `localStorage`.
- Placeholder-режимы для будущих экранов.
- GitHub Actions CI и GitHub Pages.

## Структура

- `src/data` - карты, главы, encounters и minor events.
- `src/domain` - модель состояния, трактовки, прогресс и storage.
- `src/app.ts` - рендер интерфейса и обработка действий.
- `src/style.css` - мобильная визуальная оболочка.
- `docs` - проектная документация.
- `tasks/SU` - карточки крупных задач.
- `reports` - отчёты Codex по PR.
- `.github/workflows` - CI и Pages.

## Процесс работы

1. Задача описывается в GitHub Issue.
2. Для крупной задачи создаётся файл в `tasks/SU/`.
3. Работа ведётся в feature-ветке.
4. Если задача готова к проверке, Codex открывает обычный Open PR.
5. Пользователь вручную проверяет визуальные изменения.
6. Отчёт кладётся в `reports/`.
7. PR не мержится без ручной проверки и явного решения по merge.
