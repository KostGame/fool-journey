# SU-006 - Convert journey into dialogue-based text quest

## Контекст

После SU-005 основной режим `Путь Шута` должен перейти в формат текстового квеста с диалогами. Старшие арканы воспринимаются как персонажи, главы, локации и испытания. Младшие арканы работают как способы взаимодействия с ситуацией. Дворовые арканы появляются как помощники Шута.

## Цель

Собрать качественный vertical slice диалогового квеста, чтобы пользователь мог пройти цельный эпизод в mobile-first интерфейсе, а не просто увидеть набор технических экранов.

## Что нужно сделать

- Перевести основной путь `Путь Шута` в диалоговый формат.
- Добавить минимум 5 major scenes.
- Добавить минимум 3 minor events.
- Добавить минимум 3 court helper appearances.
- Сохранить fallback для непереведённых сцен, чтобы текущий маршрут не ломался.
- Обновить UI, тексты и стили под текстовый квест.
- Обновить тесты, документацию и отчёт.

## Что не делать

- Не переписывать сразу все 22 главы.
- Не добавлять новые режимы.
- Не добавлять backend, авторизацию, внешние API или серверное хранение.
- Не превращать проект в случайный генератор гаданий.
- Не делать Draft PR.

## Критерии готовности

1. Есть `tasks/SU/SU-006-dialogue-text-quest.md`.
2. Основной путь выглядит как текстовый квест с диалогами.
3. В vertical slice есть минимум 5 major scenes, 3 minor events и 3 helper appearances.
4. Непереведённые сцены продолжают работать через fallback.
5. Локальные проверки `npm run check`, `npm test`, `npm run build`, `git diff --check` проходят.
6. Есть отчёт `reports/SU-006-dialogue-text-quest.md`.
