# SU-007 - Make journey a single-screen scene flow

## Контекст

После SU-006 и SU-006A игра всё ещё ощущается как длинная страница. На телефоне приходится проматывать вниз, чтобы найти выбор и результат. Нужен отдельный screen-by-screen flow для основного режима `Путь Шута`.

## Цель

Переделать основной режим в поток `home → scene → result → next scene`, чтобы каждая сцена и каждый результат открывались как отдельный экран на чистом листе.

## Что нужно сделать

- Разделить home screen, scene screen и result screen.
- Убрать длинную активную сцену с главной страницы.
- Сделать `Продолжить историю` входом в отдельную сцену.
- Сделать `Дальше` входом в следующую сцену из result screen.
- Сохранить progress, reset/replay и fallback-сцены.
- Обновить тесты, документацию и отчёт.

## Что не делать

- Не добавлять новые карты.
- Не добавлять новые режимы.
- Не переписывать все 22 главы.
- Не делать сложные расклады.
- Не добавлять backend, внешние API, авторизацию или базу данных.
- Не открывать Draft PR.

## Критерии готовности

1. Есть `tasks/SU/SU-007-single-screen-scene-flow.md`.
2. Home screen не содержит длинной активной сцены.
3. `Продолжить историю` открывает отдельный scene screen.
4. После выбора открывается отдельный result screen.
5. `Дальше` открывает следующую сцену сверху.
6. Progress, reset/replay и fallback работают.
7. Документация обновлена.
8. Есть отчёт `reports/SU-007-single-screen-scene-flow.md`.
