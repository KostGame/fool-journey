# SU-015: Classic text quest interface overhaul

## Контекст

Issue #43 просит приблизить `Путь Шута` к классическим текстовым квестам: локация, визуальная зона, текст сцены, действия и компактный инвентарь / дневник. После SU-014 сюжетный flow уже стабилен, но интерфейс всё ещё нужно привести к более классической форме.

## Цель

- сделать scene screen ближе к классическому текстовому квесту;
- сделать result screen похожим на результат действия, а не на лог;
- явно развести визуальную зону, текст сцены, действия и инвентарь;
- сохранить mobile-first layout без горизонтальной прокрутки;
- не ломать story flow, journal, completion и thumbnails.

## Не делаем

- не добавляем новые главы и режимы;
- не меняем сюжетную архитектуру;
- не добавляем backend, авторизацию, внешние API или серверное хранение;
- не делаем новую RPG-экономику;
- не открываем Draft PR;
- не используем слово `шпаргалка` в интерфейсе.

## Риски

- перестройка layout может случайно сделать карточки слишком узкими на 360-430 px;
- результат может снова выглядеть как лог, если текстовые блоки не разделить достаточно явно;
- нужно сохранить доступность journal и inventory без перегруза экрана.
